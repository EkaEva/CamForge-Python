## 一、現存問題

### 1.1 コードアーキテクチャ

| # | 問題 | 場所 | 深刻度 |

## 二、最適化方向

### Phase 1 — 修正と強化（v0.2）✅ 完了

- [x] **P1-1** `requirements.txt` を補完、`Pillow` 依存関係を追加
- [x] **P1-2** クロスプラットフォームフォント修正：実行時に利用可能な中国語フォントを検出、自動フォールバック
- [x] **P1-3** クロスプラットフォームスクロールホイールイベント修正：Windows / macOS / Linux の `delta` 値とイベントタイプを区別
- [x] **P1-4** ウィンドウ最大化互換性修正：プラットフォームに応じて `state('zoomed')` または `attributes('-fullscreen')` を選択
- [x] **P1-5** 浮動小数点検証問題修正：`_read_params` で角度を `int` に変換してから `validate_params` に渡す
- [x] **P1-6** `LICENSE` ファイル追加（MIT）
- [x] **P1-7** `.gitignore` を改善（IDE ディレクトリ、仮想環境、ビルド成果物等）
- [x] **P1-8** `cam_mechanics.py` の単体テスト追加（pytest、59 項すべて合格）

### Phase 2 — リファクタリングと性能（v0.3）

目標：コード構造を改善、アニメーション性能とエクスポート体験を向上。

- [ ] **P2-1** `CamSimulator` を独立モジュールに分割：
  ```
  ui/
  ├── sidebar.py      # サイドバー構築とパラメータ管理
  ├── toolbar.py      # ツールバーボタンとコントロール
  ├── plots.py        # 静的チャート描画
  ├── animation.py    # アニメーション制御とフレームレンダリング
  ├── export.py       # 画像/GIF/Excel エクスポート
  └── app.py          # メインウィンドウ組み立てと起動
  ```
- [x] **P2-2** マジックナンバーを名前付き定数または設定項目に抽出
- [x] **P2-3** GIF エクスポートをバックグラウンドスレッド + プログレスバーに移行、UI 凍結を回避
- [x] **P2-4** アニメーションと GIF エクスポート間のフォロア描画重複コードを除去、共有関数に抽出
- [x] **P2-5** 静的画像エクスポートを直接再計算描画に変更、`BlendedGenericTransform` ハックを除去
- [x] **P2-6** アニメーション終了後「再生」ボタン対応、再計算不要
- [x] **P2-7** ダウンロードファイル名を i18n キーで動的生成、言語切替に追従
- [x] **P2-8** 色定数をテーマ辞書に抽出、ダークモード準備
- [x] **P2-9** メソッド内の標準ライブラリ import をモジュールトップレベルに移動
- [x] **P2-10** `_plot_static` と `_on_download` 間の静的プロットコード重複を除去

## 四、優先度原則

1. **正確性優先** — 機能結果に影響する問題（ゼロ除算、スレッド安全、エラー処理）を体験問題より優先
2. **テスト容易性優先** — 純粋計算モジュールのテスト追加は、以降のすべてのリファクタリングの安全網
3. **漸進的リファクタリング** — 大クラス分割時に機能を保持、各ステップで独立検証可能
4. **ユーザー感知優先** — ユーザー体験に直接影響する問題（ショートカット衝突、警告上書き）を早期解決
5. **i18n 一貫性** — ユーザー可視のすべての文字列は i18n を経由、エラーメッセージとフォーマット詳細を含む
