## 二、优化方向

### Phase 1 — 修复与加固（v0.2）

目标：解决影响功能正确性和跨平台可用性的关键问题。

- [x] **P1-1** 补全 `requirements.txt`，添加 `Pillow` 依赖
- [x] **P1-2** 修复跨平台字体：运行时检测系统可用中文字体，自动回退
- [x] **P1-3** 修复跨平台滚轮事件：区分 Windows / macOS / Linux 的 `delta` 值与事件类型
- [x] **P1-4** 修复窗口最大化兼容性：按平台选择 `state('zoomed')` 或 `attributes('-fullscreen')`
- [x] **P1-5** 修复浮点校验问题：`_read_params` 中角度先转 `int` 再传入 `validate_params`
- [x] **P1-6** 添加 `LICENSE` 文件（MIT）
- [x] **P1-7** 完善 `.gitignore`（IDE 目录、虚拟环境、构建产物等）
- [x] **P1-8** 为 `cam_mechanics.py` 添加单元测试（pytest，59 项全部通过）：
  - 五种运动规律的边界值与连续性
  - 凸轮廓形闭合性（首尾坐标一致）
  - 压力角计算正确性
  - 参数校验覆盖所有分支

### Phase 2 — 重构与性能（v0.3）

目标：改善代码结构，提升动画性能与导出体验。

- [ ] **P2-1** 拆分 `CamSimulator` 为独立模块：
  ```
  ui/
  ├── sidebar.py      # 侧边栏构建与参数管理
  ├── toolbar.py      # 工具栏按钮与控件
  ├── plots.py        # 静态图表绘制
  ├── animation.py    # 动画控制与帧渲染
  ├── export.py       # 图片/GIF 导出
  └── app.py          # 主窗口组装与启动
  ```
- [x] **P2-2** 提取魔法数字为命名常量或配置项：
  ```python
  TIP_WIDTH_RATIO = 0.04
  TIP_HEIGHT_RATIO = 0.08
  ROD_LENGTH_RATIO = 4.0
  ARC_RADIUS_RATIO = 0.3
  MAX_PRESSURE_ANGLE = 30.0
  ```
- [x] **P2-3** GIF 导出改为后台线程 + 进度条，避免 UI 冻结
- [x] **P2-4** 消除动画与 GIF 导出间的推杆绘制重复代码，抽取为共享函数
- [x] **P2-5** 静态图导出改为直接重新计算绘图（而非复制 `ax.get_lines()`），消除 `BlendedGenericTransform` hack
- [x] **P2-6** 动画结束后支持「重播」按钮，无需重新计算
- [x] **P2-7** 下载文件名使用英文（displacement.png / velocity.png / cam_profile.png / cam_animation.gif）

### Phase 3 — 功能增强（v0.4）

目标：扩展仿真功能，提升工程实用性。

- [ ] **P3-1** 滚子从动件支持：增加滚子半径参数，计算实际廓形（等距曲线）与理论廓形

- [ ] **P3-4** 凸轮轮廓曲率半径计算与显示，标注最小曲率半径位置
- [ ] **P3-5** 参数预设系统：保存/加载常用凸轮配置（JSON 文件）
- [x] **P3-6** 输入框默认值填充，降低首次使用门槛
- [x] **P3-7** 键盘快捷键：Enter 开始、Space 暂停/继续/重播、R 随机
- [x] **P3-8** 静态曲线图标题标注当前运动规律名称
- [ ] **P3-9** 压力角阈值可配置（侧边栏输入框或下拉选择）

### Phase 4 — 体验升级（v0.5）

目标：打磨交互细节，提升视觉与操作体验。

- [ ] **P4-1** 深色模式支持（tkinter 主题 + matplotlib 样式切换）

- [ ] **P4-6** 多语言支持（i18n）：中/英文界面切换
- [ ] **P4-7** 导出格式扩展：支持 SVG 矢量图、CSV 数据表导出

## 四、优先级原则

1. **正确性优先** — 影响功能结果的问题（依赖缺失、浮点校验）优先于体验问题
2. **跨平台优先** — 字体与事件兼容性直接决定 macOS/Linux 用户能否正常使用
3. **可测试性优先** — 为纯计算模块添加测试是后续所有重构的安全网
4. **渐进重构** — 拆分大类时保持功能不变，每步可独立验证
5. **用户感知优先** — UI 冻结、无进度反馈等直接影响使用体验的问题尽早解决
