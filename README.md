## 项目简介

CamForge 是一款基于 Python 的凸轮机构运动学仿真桌面应用，面向机械工程教学与设计场景。支持五种经典从动件运动规律、凸轮廓形解析计算、压力角实时可视化以及全程旋转动画，帮助工程师和学生直观理解凸轮机构的运动学特性。

### 核心亮点

- **五种运动规律** — 等速、等加速等减速、简谐、摆线、五次多项式，推程与回程可独立选择
- **解析计算** — 廓形坐标与压力角均采用解析公式，非数值微分，精度高、无跳动
- **实时动画** — 360 帧旋转动画，支持速度调节、暂停/继续/重播，同步显示压力角弧线与法线/切线
- **多语言界面** — 支持中文、英文双语实时切换，导出文件名随语言自动变更
- **一键导出** — 静态图表导出为 600 DPI 高清 TIFF，动画导出为 150 DPI 循环 GIF，数据导出为 Excel 表格
- **随机参数** — 一键生成合法随机凸轮参数，快速探索不同构型
- **参数校验** — 自动检测四角之和、偏距范围等约束，异常参数即时提示
- **键盘快捷键** — Enter 开始仿真、Space 暂停/继续/重播、R 随机凸轮
- **现代化布局** — 2 行 4 列网格布局，8 个图表板块合理分布，宽屏自适应
- **高性能渲染** — NumPy 向量化计算，主题切换控件缓存，动画流畅无卡顿

### 运动规律一览

| 编号 | 运动规律 | 推程位移公式 | 特性 |
|:----:|---------|------------|------|
| 1 | 等速运动 | $s = h \cdot \dfrac{\delta}{\delta_0}$ | 速度恒定，行程始末有刚性冲击 |
| 2 | 等加速等减速 | $s = 2h\left(\dfrac{\delta}{\delta_0}\right)^2$（前半段） | 加速度恒定，行程中点有柔性冲击 |
| 3 | 简谐运动 | $s = \dfrac{h}{2}\left(1 - \cos\dfrac{\pi\delta}{\delta_0}\right)$ | 无刚性冲击，行程始末有柔性冲击 |
| 4 | 摆线运动 | $s = h\left(\dfrac{\delta}{\delta_0} - \dfrac{1}{2\pi}\sin\dfrac{2\pi\delta}{\delta_0}\right)$ | 无刚性冲击和柔性冲击，动力性能好 |
| 5 | 五次多项式 | $s = h\left(10t^3 - 15t^4 + 6t^5\right)$ | 高阶平滑，综合性能最优 |

### 动画特性

- **反转法**：凸轮旋转，从动件固定，直观展示实际运动
- **固定铰支座**：旋转中心绘制标准固定铰支座符号（铰链圆圈 + 三角形 + 底座阴影线）
- **压力角弧线**：在接触点实时绘制压力角弧线与标注
- **法线/切线**：解析计算廓形导数方向，精确绘制接触点法线与切线
- **相位分界线**：径向标注推程/远休止/回程/近休止四个阶段
- **信息面板**：实时显示当前转角、压力角、位移、行程、初始位移
- **可配置显示**：8 项动态显示选项（切线、法线、压力角弧线、角度分界线、基圆、偏距圆、推杆上下限、网格线），导出 GIF 与界面显示完全一致

## 使用指南

### 1. 语言切换

侧边栏顶部选择语言（中文 / English），界面所有标签、按钮、图表标题、状态提示、导出文件名即时切换。

### 2. 参数输入

在左侧面板中依次填入凸轮设计参数：

| 参数 | 符号 | 说明 | 约束 |
|-----|:----:|------|------|
| 推程运动角 | $\delta_0$ | 推程阶段凸轮转角 | 整数，> 1° |
| 远休止角 | $\delta_{01}$ | 从动件停留在最远位置 | 整数，> 1° |
| 回程运动角 | $\delta_{0}'$ | 回程阶段凸轮转角 | 整数，> 1° |
| 近休止角 | $\delta_{02}$ | 从动件停留在最近位置 | 整数，> 1° |
| 行程 | $h$ | 从动件最大位移 | > 0 mm |
| 基圆半径 | $r_0$ | 凸轮基圆半径 | > 0 mm |
| 偏距 | $e$ | 从动件导路偏移量 | < $r_0$ mm |
| 角速度 | $\omega$ | 凸轮旋转角速度 | > 0 rad/s |

> 四个角度之和必须恰好等于 **360°**。

### 3. 选择运动规律

- **推程规律**：下拉选择 1-5 号运动规律
- **回程规律**：下拉选择 1-5 号运动规律
- **旋向**：顺时针 / 逆时针
- **偏距方向**：正偏距 / 负偏距

### 4. 运行仿真

1. 点击 **「开始」** 按钮（或按 Enter），计算并绘制静态曲线与凸轮廓形
2. 动画自动开始播放，展示凸轮旋转过程
3. 使用 **「暂停」** 按钮暂停/继续动画（或按 Space）
4. 动画结束后按 Space 或点击 **「重播」** 重新播放
5. 拖动 **速度滑块** 调节动画帧率（1 = 最慢，10 = 最快）

### 5. 动态显示选项

侧边栏"动态显示"区域提供 8 个可选项，控制动画视图中辅助元素的显示：

| 选项 | 说明 | 默认 |
|------|------|:----:|
| 切线 | 接触点切线 | 关 |
| 法线 | 接触点法线 | 关 |
| 压力角弧线 | 压力角弧线与参考虚线 | 关 |
| 角度分界线 | 四阶段径向分界 | 关 |
| 基圆 | 基圆轮廓 | 关 |
| 偏距圆 | 偏距圆轮廓 | 关 |
| 推杆上下限 | 推杆行程上下限水平线 | 关 |
| 网格线 | 坐标网格 | 关 |

> 导出 GIF 时，动态显示选项与界面完全一致——勾选的元素才会出现在导出图中。

### 6. 导出结果

勾选需要导出的内容，点击 **「下载」** 按钮：

| 类型 | 格式 | 说明 |
|------|------|------|
| 位移曲线 | TIFF (600 DPI) | 推杆位移随转角变化 |
| 速度曲线 | TIFF (600 DPI) | 推杆速度随转角变化 |
| 加速度曲线 | TIFF (600 DPI) | 推杆加速度随转角变化 |
| 凸轮廓形 | TIFF (600 DPI) | 凸轮廓形 + 基圆 + 偏距圆 + 固定铰支座 |
| 动态图 | GIF (150 DPI) | 360 帧循环动画，含进度提示 |
| Excel | XLSX | 转角 δ、向径 R、推杆速度 v、推杆加速度 a |

### 7. 随机探索

点击 **「随机凸轮」** 按钮（或按 R），自动生成一组满足约束的随机参数，快速体验不同凸轮构型。

### 设计原则（v0.2）

- **组合模式**：CamSimulator 通过组合委托各管理器，职责清晰，易于维护
- **计算与展示分离**：`cam_mechanics.py` 为纯数学库，不依赖任何 GUI 框架
- **NumPy 向量化**：`compute_roller_profile` 和 `compute_curvature_radius` 使用 `np.roll` 替代 Python 循环，性能提升 10 倍 +
- **解析优先**：所有导数与压力角均采用解析公式（`ds/dδ = v/ω`），避免数值微分带来的精度损失
- **1° 步长离散**：全程 360 个离散点，每个整数角度对应一个计算点，兼顾精度与性能
- **国际化设计**：所有 UI 字符串通过 i18n 键值查找，运行时切换语言无需重启
- **控件缓存**：主题切换时遍历缓存列表而非递归遍历控件树，性能提升 5 倍 +
- **ParameterModel**：类型安全的参数模型，替代裸 dict 传递，支持校验与转换

## 核心算法

### 凸轮廓形计算（反转法）

廓形坐标由反转法解析求得：

$$
x = -s_n \left[ (s_0 + s)\sin\delta + p_z \cdot e\cos\delta \right]
$$

$$
y = (s_0 + s)\cos\delta - p_z \cdot e\sin\delta
$$

其中 $s_0 = \sqrt{r_0^2 - e^2}$，$s_n = \pm 1$ 为旋向符号，$p_z = \pm 1$ 为偏距符号。

### 压力角计算

$$
\alpha = \arctan\!\left(\frac{ds/d\delta - p_z \cdot e}{s_0 + s}\right)
$$

当最大压力角超过 **30°** 时，界面将显示红色警告提示。

## 常见问题

<details>
<summary><b>启动后界面中文显示为方块</b></summary>

本程序自动检测系统可用字体并回退。若仍有问题：
- **Windows**：一般自带，无需处理
- **macOS**：程序会自动检测 Yu Gothic / Hiragino Sans 等字体
- **Linux**：`sudo apt install fonts-wqy-microhei` 并修改字体配置

</details>

<details>
<summary><b>动画播放卡顿</b></summary>

- 降低速度滑块档位
- 关闭不需要的显示选项（法线/切线、网格等）
- 动画默认每 2 帧刷新一次画布以减少延迟

</details>

<details>
<summary><b>GIF 导出时间较长</b></summary>

GIF 导出需逐帧渲染 360 帧图像，采用后台线程 + 进度条避免界面冻结。帧渲染完成后还需合成 GIF，进度条会提示"正在合成GIF..."。请耐心等待。

</details>
